# Configuration Reference

Environment variables and config files for Relay.

## Core `.env` variables

| Variable | Default | Description |
| --- | --- | --- |
| `APP_KEY` | — | Laravel app key. Generated by `composer setup`. |
| `APP_URL` | `http://localhost` | Base URL used in OAuth callbacks. |
| `DB_CONNECTION` | `sqlite` | Database driver. SQLite is the supported default for local-first deployments. |
| `QUEUE_CONNECTION` | `database` | Queue driver. Tests override to `sync`. |
| `BROADCAST_CONNECTION` | `log` | Event broadcasting driver. UI uses polling, not WebSockets. |

## AI provider — `config/ai.php`

| Variable | Default | Description |
| --- | --- | --- |
| `AI_PROVIDER` | `anthropic` | Default provider. One of `anthropic`, `openai`, `gemini`, `claude_code_cli`. |
| `ANTHROPIC_API_KEY` | — | Required for Anthropic provider. |
| `ANTHROPIC_MODEL` | `claude-sonnet-4-6` | Model ID. |
| `ANTHROPIC_BASE_URL` | `https://api.anthropic.com` | API endpoint. |
| `OPENAI_API_KEY` | — | Required for OpenAI provider. |
| `OPENAI_MODEL` | `gpt-4o` | Model ID. |
| `OPENAI_BASE_URL` | `https://api.openai.com` | API endpoint. |
| `GEMINI_API_KEY` | — | Required for Gemini provider. |
| `GEMINI_MODEL` | `gemini-2.5-flash` | Model ID. |
| `GEMINI_BASE_URL` | `https://generativelanguage.googleapis.com` | API endpoint. |
| `CLAUDE_CODE_COMMAND` | `claude --dangerously-skip-permissions --print --output-format stream-json` | Base command line for Claude Code invocations. Additional args (`--model`, `--allowedTools`, prompt) are appended by the provider. |
| `CLAUDE_CODE_WORKING_DIR` | — | Optional working directory override. |
| `CLAUDE_CODE_TIMEOUT` | `300` | Timeout in seconds. |

Provider selection is resolved via a scope cascade: workspace+stage → workspace → global+stage → global → `AI_PROVIDER` default. Overrides are persisted in the `provider_configs` table and configurable via the `/config` UI.

## Pipeline — `config/relay.php`

| Variable | Default | Description |
| --- | --- | --- |
| `RELAY_ITERATION_CAP` | `5` | Maximum Verify → Implement bounces before a run enters `StuckState::IterationCap`. |
| `RELAY_CHANGELOG_PATH` | `CHANGELOG.md` | Path to the changelog updated by the Release agent. |
| `RELAY_DEPLOY_HOOK` | — | Optional URL fired by the Release agent's `trigger_deploy` tool after PR merge. |

### Mobile (NativePHP)

| Variable | Default | Description |
| --- | --- | --- |
| `RELAY_MOBILE_PLATFORM` | — | Platform hint for mobile builds. |
| `RELAY_MOBILE_NETWORK` | `wifi` | Network status reported to sync scheduling. |
| `RELAY_MOBILE_LOW_POWER` | `false` | Low-power mode flag. |
| `RELAY_MOBILE_WIFI_SYNC_INTERVAL` | `5` | Sync interval (minutes) on Wi-Fi. |
| `RELAY_MOBILE_CELLULAR_SYNC_INTERVAL` | `15` | Sync interval (minutes) on cellular. |
| `RELAY_MOBILE_SYNC_INTERVAL` | `5` | Fallback sync interval (minutes). |
| `RELAY_MOBILE_OAUTH_HOST` | `127.0.0.1` | OAuth callback host for mobile. |
| `RELAY_MOBILE_OAUTH_PORT` | `8100` | OAuth callback port for mobile. |

## OAuth — `config/services.php`

| Variable | Default | Description |
| --- | --- | --- |
| `GITHUB_CLIENT_ID` | — | GitHub OAuth app client ID. |
| `GITHUB_CLIENT_SECRET` | — | GitHub OAuth app client secret. |
| `GITHUB_REDIRECT_URI` | `http://localhost:8000/oauth/github/callback` | OAuth callback URL. Must match the GitHub app configuration. |
| `JIRA_CLIENT_ID` | — | Jira OAuth app client ID. |
| `JIRA_CLIENT_SECRET` | — | Jira OAuth app client secret. |
| `JIRA_REDIRECT_URI` | `http://localhost:8000/oauth/jira/callback` | OAuth callback URL. |

GitHub OAuth scopes: `repo`, `read:org`, `workflow`. Jira OAuth scopes: `read:jira-work`, `write:jira-work`, `read:jira-user`, `offline_access`.

## Config files

| File | Purpose |
| --- | --- |
| `config/ai.php` | AI provider definitions and default. |
| `config/relay.php` | Iteration cap, changelog path, deploy hook, mobile sync. |
| `config/services.php` | OAuth credentials (GitHub, Jira) and Slack bot. |
| `config/nativephp.php` | NativePHP build and updater config. |

## Database-persisted config

Some settings live in tables, not config files, because they are scope-aware:

| Table | Keyed by | Managed via |
| --- | --- | --- |
| `autonomy_configs` | `scope`, `scope_id`, `stage` | `/config` page and issue detail page |
| `escalation_rules` | `id` (ordered list) | `/config` page |
| `provider_configs` | `scope`, `stage` | `/config` page |
| `filter_rules` | `source_id` | `/intake/sources/{source}/rules` |
